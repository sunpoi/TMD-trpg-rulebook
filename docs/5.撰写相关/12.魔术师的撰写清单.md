# 12.魔术师的撰写清单

魔术师的撰写清单

笔者的话：  
本文主要用途为将撰写中约定俗成的规则或常见问题记录于纸面上，并说明为什么不支持某些写法的原因，以便供撰写者对【计算正确】以外的部分进行参考。  
这部分内容将会持续更新。

1.添油算法  
添油算法是指在某一下级能力已使用动作的情况下，通过对该能力升级的方式获得DP，并继续投入到该动作之中获取更多DP。  
对于添油算法，笔者的推荐：尽可能少用以防止滥用。  
例如某能力具有3-4个等级，则只在第一等级和最后一等级使用动作，其余阶段仅使用常态DP增加该能力效果。

2.购买高级特性  
对于高等级资源而言，购买高等级特性本身不是一种问题。但不应集中选择：每轮一次，令一次攻击造成的伤害增加一倍/每轮一次，将近战/远程伤害减少至原先的1/4/免疫某伤害性质伤害/无敌/复活/即死等此类特性进行大量购买。  
诚然，此类特性是价值最高，最值得购买的特性，但这样的资源一旦泛滥，就会挤压同等不购买此类特性资源的生存空间。  
说到底，跑团是数值游戏。因此，笔者的推荐：尽可能少在高等级中复数购买上述特性，尽可能少对上述特性进行特性升级（如每影片一次无敌升级为一轮一次无敌等）。

3.体系全能  
体系全能是特指在某一资源中对本书中记录的特性和写法大规模使用，最终达到全能或接近全能的情况。  
例如在资源中各个角落分别购买某来源免疫，只要一直购买，最终能够组成接近于全来源免疫或全来源免疫。  
或对于某一能力的强化特性（如冲锋），在某一资源内购买所有或几乎所有和该能力相关的特性。  
或对于高级特性（如清单条目2中所列举的），在某一资源内购买所有或几乎所有和该能力相关的特性。  
笔者的推荐：如果没活就不要硬整，把所有东西都买一遍是过不了审的。

4.环团特供  
特指在成长方面专为环团而写的资源，把所有的成长性能力都写成最多成长4次。  
笔者的推荐：这是最恶劣的一种撰写行为，如出现此类资源就不必投稿了。

5.成长嵌套  
指将影片成长、购买成长、连续技等条件成长进行嵌套使用。  
如先使用影片成长/购买成长获取更多DP，然后投入到连续技中作为基础赚取DP，再使用动作/难易权值作为杠杆获得大量DP。  
笔者的推荐：不使用此类写法。
